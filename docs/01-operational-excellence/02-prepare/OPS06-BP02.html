OPS06-BP02: デプロイメントのテスト

本番環境と同じデプロイメント構成、セキュリティ制御、手順を使用して、プリプロダクション環境でリリース手順をテストします。

公式ドキュメント

リスクレベル

高リスク

このベストプラクティスを確立しない場合のリスクレベル

  • 本番環境でのデプロイメント問題
  • ユーザーによる問題報告まで検出不可
  • 証明書の誤インストール
  • 頻繁なトラブルシューティングとエスカレーション
テストされていないデプロイメントは、
本番環境で頻繁に問題を引き起こし、
ユーザー影響が発生します！

望ましい結果

  • テスト駆動開発文化の採用
  • リリース管理よりもビジネス価値に集中
  • デプロイメントリスクの早期特定
  • 適切な緩和策の決定
  • 本番リリースへの信頼性向上
プリプロダクションでの徹底的なテストにより、
本番環境への影響を最小化！

実装ガイダンス

  • 本番環境ミラーリング：可能な限り本番に近い環境
  • デプロイメント検証：ファイル、設定、サービス確認
  • 包括的テスト：機能、統合、負荷テスト
  • ヘルスチェック：監視とヘルスチェック
  • IaC活用：Infrastructure as Codeによる自動化
デプロイメントプロセスのテストは、
変更のテストと同じくらい重要！

実装ステップ

  • プリインストールチェック：ドリフト検出と変更セット
  • 手動承認ステップ：AWS CodePipelineでの承認
  • デプロイメント設定：AWS CodeDeploy AppSpecファイル
  • サービス統合：他のAWSサービスとの統合
  • 自動テスト実行：機能、セキュリティ、回帰テスト
  • 監視とアラート：CloudWatchによる監視
CI/CDパイプラインに組み込まれた
自動テストで品質保証！

主要AWSサービス

  • AWS CodePipeline：CI/CDパイプライン
  • AWS CodeDeploy：自動デプロイメント
  • AWS CloudFormation：Infrastructure as Code
  • Amazon CloudWatch：監視とアラート

関連ドキュメント

  • AWS CodeDeploy ユーザーガイド
  • AWS CodePipeline ユーザーガイド
  • AWS CloudFormation ドリフト検出
  • AWS CloudFormation 変更セット
