OPS07-BP04: プレイブックを使用した問題調査

プレイブックはインシデントを調査するために使用されるステップバイステップガイドです。調査、影響範囲の特定、根本原因の特定に使用されます。

公式ドキュメント

リスクレベル

中リスク

このベストプラクティスを確立しない場合のリスクレベル

  • インシデント調査の標準方法不在
  • 記憶や経験に依存したトラブルシューティング
  • 新メンバーの試行錯誤による学習
  • 調査ベストプラクティスのチーム間共有不足
標準的な調査方法がないと、
インシデント対応が非効率になり、
復旧時間が延長します！

望ましい結果

  • 一般的なインシデント用のプレイブック保有
  • 中央保存とチームメンバーへのアクセス提供
  • プレイブックの頻繁な更新
  • 既知の根本原因へのコンパニオンランブック構築
  • チーム全体での一貫した調査手法
プレイブックにより、
インシデント軽減の取り組みを強化し、
復旧時間を短縮！

優れたプレイブックの特徴

  • ステップバイステップ：発見プロセスのガイド
  • ツール・権限：必要なリソースの明確化
  • コミュニケーション計画：ステークホルダー更新
  • エスカレーション計画：根本原因不明時の対応
  • ランブック連携：解決方法への指針
  • アラート統合：特定アラートからの直接アクセス
多くの場合、プレイブックが根本原因を特定し、
ランブックで軽減策を実行！

実装ステップ

  • インシデント特定：一般的なインシデントのリスト作成
  • 低リスクから開始：シンプルなシナリオから構築
  • テキスト形式：中央ドキュメントリポジトリでの管理
  • 段階的自動化：Python等のスクリプト言語活用
  • AWS統合：Systems Manager Automationの活用
  • 完全自動化：イベント応答による自動実行
組織の成熟度に応じて、
テキストから完全自動化まで進化！

インシデント管理サービス

  • AWS Systems Manager Incident Manager：統合インシデント対応
  • AWS Systems Manager Automation：プレイブック自動化
  • Jupyter Notebook：半自動化スクリプト実行
  • Collaboration Tools：チーム連携機能
