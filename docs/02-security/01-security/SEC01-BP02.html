SEC01-BP02 アカウントルートユーザーとプロパティを保護する

ルートユーザーは最も特権的なユーザーであり、適切な制御を確立し、日常的な使用を避けることでリスクを軽減します。

期待される成果

ルートユーザー認証情報の誤用による意図的・偶発的な損害の可能性を軽減し、ルートユーザー使用時の適切な担当者への警告を提供します。

一般的なアンチパターン

  • ルート認証情報を必要とする以外のタスクでルートユーザーを使用する
  • 緊急時の重要インフラ・プロセス・担当者の機能を定期的にテストしない
  • 代替アカウント復旧方法を考慮・テストしない
  • DNS・メールサーバー・電話プロバイダーをセキュリティ境界として扱わない
