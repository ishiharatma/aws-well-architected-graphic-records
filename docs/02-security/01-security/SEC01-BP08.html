通知とサブスクリプション

Amazon SNSトピック
  • AWS Daily Feature Updatesトピックを購読して最新情報の包括的なサマリーを確認
  • Amazon GuardDutyやAWS Security Hubなどのセキュリティサービス固有のSNSトピックを購読
  • 各サービスに関する新しい標準、検出結果、その他の最新情報を入手
RSSフィード
  • お気に入りのRSSリーダーでさまざまなブログや速報を購読
  • 各ブログのカテゴリ別フィードも利用可能
メールサブスクリプション
  • 各種ニュースレターやアップデート情報をメールで受信
  • AWS アカウントセンターからサブスクリプション設定が可能
SNSトピックを購読すれば

AWSイベントとカンファレンス

AWS主催のカンファレンス、イベント、ウェビナーでは、新しいサービスや機能が発表され、詳しく説明されています。

AWS re:Inforce

毎年開催されるAWS専門のセキュリティカンファレンス。最新のセキュリティ機能やベストプラクティスに特化。

AWS re:Invent

年次の大規模カンファレンス。新サービスや機能の発表が多数行われる。セキュリティ関連のセッションも充実。

AWS Webinars

定期的に開催されるオンラインセミナー。特定のセキュリティトピックに焦点を当てた詳細な解説が視聴可能。

AWS Events YouTube

YouTube の AWS Events チャンネルでは、過去のカンファレンスセッションやウェビナーを視聴可能。

AWSアカウントチームとの連携

  • セキュリティサービスの最新情報や新しい推奨事項についてAWS アカウントチームに問い合わせ

  • AWS エンタープライズサポート登録でTechnical Account Manager (TAM)から毎週最新情報を入手
  • TAMとの定期的なレビューミーティングを設定可能

  • 新機能の早期アクセスやパイロット参加の機会
  • 重要なセキュリティイベントに関する優先通知

実装手順

1. 情報源のサブスクリプション設定

  • お気に入りのRSSリーダーでさまざまなブログや速報を購読
  • 「Daily Feature Updates」SNSトピックを購読
  • セキュリティ関連サービスの専用SNSトピックを購読

2. AWSイベント参加計画

  • 新しい機能やサービスについて直に学ぶため、どのAWSイベントに参加すべきか評価
  • 参加できない場合はオンラインセッションや録画を視聴

3. アカウントチームとの連携

  • セキュリティサービスや機能の更新についてAWS アカウントチームとのミーティングを設定
  • 最新のセキュリティ推奨事項やベストプラクティスについて相談

4. サポートレベルの検討

  • エンタープライズサポートへの登録を検討
  • 登録すると、Technical Account Manager (TAM)に定期的に相談可能
  • TAMとの定期レビューで最新の推奨事項を入手

5. 評価と実装プロセスの確立

  • 新しいセキュリティサービスや機能を評価するプロセスを確立
  • 環境への影響評価とリスク分析を実施
  • テスト環境での試験を経て本番環境に実装
情報収集→評価→実装の

組織内での情報共有

新しいセキュリティサービスや機能の情報を組織内で効果的に共有するための方法：

定期的な情報共有の仕組み
  • セキュリティ関連の社内ニュースレターの発行
  • 新機能や注目のアップデートに関する社内勉強会の開催
  • セキュリティ関連情報を共有するための専用チャットチャネルの設置
チーム体制と責任の明確化
  • 最新情報の収集と共有を担当する責任者の指定
  • 新機能の評価担当者の明確化
  • 実装判断の承認フローの確立
継続的な学習の奨励
  • AWS認定の取得支援
  • AWS Skill Builderなどの学習リソースの活用
  • 社内での知識共有文化の醸成
