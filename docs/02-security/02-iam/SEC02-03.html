SEC02-03 アイデンティティとアクセス管理

適切なアイデンティティ管理とアクセス制御により、正しいユーザーが適切なリソースに適切な条件下でアクセスできることを確保します。 認証（SEC02）と権限管理（SEC03）の両方を包含します。

公式ドキュメント

SEC02-03 統計情報

15 ベストプラクティス
12 高リスク
3 中リスク
0 低リスク

SEC02

認証

人とマシンの認証を管理し、適切なアイデンティティ検証を実装します。

6つのベストプラクティス

SEC03

権限管理

人とマシンの権限を管理し、最小権限の原則を実装します。

9つのベストプラクティス

SEC02: 人とマシンの認証管理

SEC02-BP01

強力なサインイン機構を使用する

強力な認証機構を実装し、多要素認証（MFA）を使用してアカウントのセキュリティを強化します。

高リスク

SEC02-BP02

一時的な認証情報を使用する

長期的な認証情報の代わりに一時的な認証情報を使用し、セキュリティリスクを軽減します。

高リスク

SEC02-BP03

シークレットを安全に保存し使用する

API キー、パスワード、証明書などのシークレットを安全に保存し、適切にアクセス制御します。

高リスク

SEC02-BP04

中央集約型アイデンティティプロバイダーに依存する

中央集約型のアイデンティティプロバイダーを使用して、一貫したアイデンティティ管理を実現します。

中リスク

SEC02-BP05

認証情報を定期的に監査しローテーションする

認証情報の使用状況を定期的に監査し、不要な認証情報を削除し、必要に応じてローテーションします。

中リスク

SEC02-BP06

ユーザーグループと属性を活用する

ユーザーグループと属性を使用して、効率的で一貫したアクセス管理を実現します。

中リスク

SEC03: 人とマシンの権限管理

SEC03-BP01

アクセス要件を定義する

人とマシンのアイデンティティに対するアクセス要件を明確に定義し、適切な権限管理を実現します。

高リスク

SEC03-BP02

最小権限アクセスを付与する

必要最小限の権限のみを付与し、セキュリティリスクを最小化します。

高リスク

SEC03-BP03

緊急アクセスプロセスを確立する

緊急時に必要なアクセスを迅速に提供するプロセスを確立します。

高リスク

SEC03-BP04

権限を継続的に削減する

不要な権限を定期的に見直し、継続的に削減します。

高リスク

SEC03-BP05

組織の権限ガードレールを定義する

組織全体で一貫した権限管理を実現するガードレールを定義します。

高リスク

SEC03-BP06

ライフサイクルに基づいてアクセスを管理する

ユーザーのライフサイクルに応じてアクセス権限を適切に管理します。

高リスク

SEC03-BP07

パブリックアクセスとクロスアカウントアクセスを分析する

パブリックアクセスとクロスアカウントアクセスを定期的に分析し、リスクを評価します。

高リスク

SEC03-BP08

組織内でリソースを安全に共有する

組織内でのリソース共有を安全に実行するメカニズムを実装します。

高リスク

SEC03-BP09

第三者とリソースを安全に共有する

第三者とのリソース共有を安全に実行するプロセスを確立します。

高リスク
