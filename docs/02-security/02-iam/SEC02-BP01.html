SEC02-BP01 強力なサインイン機構を使用する

多要素認証（MFA）と強力なパスワードポリシーを要求することで、サインイン認証情報が意図せず開示されたり推測されやすい場合のリスクを軽減します。

公式ドキュメント

リスクレベル

高リスク

このベストプラクティスを確立しない場合のリスクレベルです。

適切な実装で
リスク軽減！

期待される成果

IAMユーザー、AWSアカウントルートユーザー、AWS IAM Identity Center、サードパーティアイデンティティプロバイダーに対して強力なサインイン機構を使用することで、AWSの認証情報への意図しないアクセスのリスクを軽減します。

明確な
目標設定！

一般的なアンチパターン

以下のパターンは避けるべき設計です：

  • 複雑なパスワードとMFAを含む強力なパスワードポリシーを適用しない
  • 異なるユーザー間で同じ認証情報を共有する
  • 疑わしいサインインに対する検出制御を使用しない
これらの落とし穴を
回避しよう！

実装方法

  • 中央集約型アイデンティティプロバイダーとのフェデレーション
  • AWS IAM Identity Center（ワークフォースユーザー向け）
  • Amazon Cognito（顧客アイデンティティ管理向け）
  • AWS Verified Access（追加セキュリティレイヤー）
複数の
実装選択肢！

IAM Identity Center推奨事項

  • 事前定義されたパスワードポリシー
  • コンテキスト認識またはalways-on MFA設定
  • ユーザーによる自身のMFAデバイス登録許可

最適化！

  • パスワード強度設定の構成
  • ユーザーへのMFA要求
  • 適応認証による疑わしいサインインのブロック

MFA要件

  • MFAの必須化 - 追加セキュリティレイヤー
  • 最小パスワード長の適用
  • パスワード複雑性の適用
  • 個別アイデンティティの作成（共有認証情報の回避）
