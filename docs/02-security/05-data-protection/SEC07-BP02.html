SEC07-BP02 データの機密性に基づいてデータ保護統制を適用する

2025年4月10日

概要

  • データの分類ポリシーで定義されたレベルに適した保護統制を適用
  • データの可用性と使用を確保しながら、機密データを不正アクセスから保護
  • リスクと機密性に応じた適切なレベルの保護を実現
  • データの保管・取り扱いに関する明確なガイドラインを提供
すべてのデータに同じ保護レベルは
非効率的！データの重要度に合わせて
適切な保護を適用しよう！

期待される成果

  • 明確なデータ分類ポリシーが定義されている
  • 分類レベルごとに承認された保管場所と必要な構成のガイドラインが存在
  • データ保護レベルとコストのバランスが取れている
  • 許可されていない場所や環境でのデータ処理を検知し警告する監視体制
明確なガイドラインと監視体制で
コンプライアンス違反を未然に防止！
見えないところにリスクは潜むよ

一般的なアンチパターン

  • すべてのデータに同じ保護レベルを適用し、一部データの過剰保護や不十分な保護を引き起こす
  • データ保護統制の定義に関係者が参加していない（セキュリティ・コンプライアンス・ビジネス部門）
  • 保護統制の運用コストとオーバーヘッドを見落としている
  • 保護統制の定期的なレビューを実施していない
  • 保存中・転送中のデータの完全なインベントリがない
全部同じ保護レベルじゃダメ！
定期的な見直しと関係者の協力が
効果的な保護には欠かせないよ

このベストプラクティスを活用するメリット

  • 重要データに適切なリソースを集中投資可能（保護、監視、修復など）
  • 機密性の低いデータに対するアクセシビリティの向上
  • データ保護要件を順守しながら柔軟な活用が可能
  • データの完全性が向上し、従業員・顧客・関係者の満足度向上
重要なデータには多くのリソースを、
そうでないものには最小限の保護を。
コスト最適化につながるよ！

データ機密レベルの例

公開 (Public)

一般公開可能なデータ。製品情報、公開ドキュメント、プレスリリースなど。

内部 (Internal)

組織内での使用を意図したデータ。社内連絡、社内マニュアル、トレーニング資料など。

機密 (Confidential)

保護が必要な内部データ。顧客情報、財務情報、人事データなど。

制限 (Restricted)

最高レベルの保護が必要な機密データ。個人識別情報(PII)、知的財産、取引機密など。

分離境界の定義

  • AWS アカウント分離: 機密レベルごとに別アカウントでデータを管理
  • IAM権限: 最小特権の原則に基づくアクセス制御
  • サービスコントロールポリシー(SCP): 許可されるサービスとアクションの制限
  • ネットワーク分離: VPC、サブネット、セキュリティグループによる分離
  • データ保管の隔離: 機密レベルに応じた保存場所の分離
アカウント分離は強固な境界を作る
最も効果的な方法！
最小特権とSCPで制御を強化しよう

保護統制の実装

保存データの保護
  • 暗号化: KMS、S3暗号化、EBS暗号化
  • アクセス制御: IAM、リソースポリシー
  • 監査ログ: CloudTrail、CloudWatch
  • データライフサイクル: S3ライフサイクル、バックアップ
転送中データの保護
  • 転送暗号化: TLS/SSL、VPN
  • 安全な接続: PrivateLink、VPC Endpoint
  • ネットワーク制御: NACLs、セキュリティグループ
  • トラフィック監視: VPC Flow Logs
機密レベル低減技術
  • トークン化: 機密データを代替値に置き換え
  • データマスキング: 一部データを隠蔽
  • 匿名化: 個人識別情報の除去
  • 仮名化: 間接的な識別子への置き換え
データの機密レベルを下げる技術で
活用範囲を広げつつ、
コンプライアンスも守れるよ！

監視と検証

  • 継続的なモニタリング: 保護統制の有効性を常に確認
  • 自動化されたテスト: コンプライアンス違反の自動検出
  • アラート設定: 異常なデータアクセスや移動の検知
  • 監査ログ: すべてのデータアクセスと変更を記録
  • 定期的なレビュー: 保護統制とポリシーの見直し
実装して終わりじゃないよ！
継続的な監視と定期的な見直しで
新たな脅威からも守ろう

実装手順

1. データの分類と機密レベルの特定

  • ワークロード内のすべてのデータタイプを特定
  • 組織のポリシーに基づいて分類レベルを割り当て
  • データの所在と処理場所のマッピング

2. 分離境界の定義と実施戦略の決定

  • アカウント分離、VPC分離などの境界を決定
  • SCPやIAMポリシーによるアクセス制御戦略を策定
  • 異なる環境間のデータ移動ルールを定義

3. 保護統制の評価と実装

  • 必要な暗号化、アクセス制御、監査メカニズムを特定
  • 各分類レベルに適切な保護統制を適用
  • 保護統制の技術的実装を行う

4. 機密レベル低減オプションの評価

  • トークン化、匿名化など機密レベルを下げる方法を検討
  • 必要に応じてトークン化サービスなどを導入
  • データ変換プロセスを定義し実装

5. 監視と検証の自動化

  • コンプライアンス違反の検出メカニズムを構築
  • アラートと通知プロセスを確立
  • 定期的な監査とレポート生成を自動化

  • 教育とトレーニング: 従業員への分類ポリシーの周知と責任の理解

  • 文書化: 明確なポリシーと手順の文書化
  • 多層防御: 複数の保護層による防御体制
