SEC09-BP02 伝送中に暗号化を適用する

2025年4月10日

概要

  • 組織的、法的、コンプライアンス要件を満たすための暗号化
  • 機密データをVPCの外部に送信する場合は暗号化プロトコルのみを使用
  • 信頼できないネットワークを転送中もデータの機密性を保持
  • AWSリソース間、およびリソースとインターネット間の通信を暗号化
データが移動する時こそ
守る必要があるんだ！
暗号化は基本中の基本だよ

期待される成果

  • リソースとインターネット間のネットワークトラフィックを暗号化
  • セキュリティ要件に従って内部AWS環境内のネットワークトラフィックも暗号化
  • 転送中のデータはすべて安全なTLSプロトコルと暗号スイートを使用
  • データへの不正アクセスを軽減
内部通信も暗号化すると
万が一の侵入時にも
データを守れるよ！

一般的なアンチパターン

  • 廃止されたバージョンのSSL、TLS、暗号スイートを使用（SSL v3.0、1024-bit RSA、RC4暗号）
  • パブリック向けリソースとの間で暗号化されていない通信（HTTP）を許可
  • X.509証明書をモニタリングせず、期限切れ前に交換しない
  • TLSに自己署名X.509証明書を使用
暗号化するだけじゃダメ！
最新の安全な方式を使って

AWS暗号化サービス

  • AWS Certificate Manager (ACM): パブリックTLS証明書の管理とデプロイ
  • AWS Private CA: プライベート認証局の作成と管理
  • CloudFront: エッジでのHTTPSリダイレクトと最新の暗号化
  • Application Load Balancer: HTTPからHTTPSへのリダイレクト
  • Security Groups: HTTPトラフィックのブロック
  • VPN: オンプレミスネットワークとAWS間の安全な接続
ACMを使えば証明書の

実装のポイント

AWSエンドポイント

AWSのサービスにはHTTPSエンドポイントが用意されており、通信にTLSを使用します。AWS APIとの通信では常にHTTPSを使用しましょう。

HTTPのブロック
  • セキュリティグループでHTTPプロトコルをブロック
  • CloudFrontまたはALBでHTTPをHTTPSに自動リダイレクト
  • S3バケットポリシーでHTTP経由のアップロードを制限
VPN接続

外部ネットワークまたはAWS Direct ConnectからVPCにVPNで接続することで、トラフィックの暗号化を円滑にできます。

TLSバージョン注意

AWS APIを呼び出す際は少なくともTLS 1.2を使用してください。TLS 1.3の使用を推奨します。

セキュリティグループで
HTTPを許可しないように
設定するのが基本だよ！

証明書管理

AWS Certificate Manager
  • AWSサービスで使用するパブリックTLS証明書の管理
  • 証明書の自動更新でセキュリティを維持
  • CloudFront, ALB, API Gateway等との簡単な統合
  • 追加料金なしで使用可能
AWS Private Certificate Authority
  • プライベート認証局の構築と管理
  • 組織内での内部使用に適した証明書の発行
  • マイクロサービス間の相互TLS認証に最適
  • 証明書のライフサイクル管理を自動化
証明書の注意点
  • 自己署名証明書は本番環境では使用しない
  • 証明書の有効期限を監視する仕組みを構築
  • 使用する暗号アルゴリズムが最新かつ強力なものか確認
  • 証明書のプライベートキーを適切に保護
証明書の期限切れは
サービス停止の原因に！
ACMなら自動更新で安心だね

Amazon S3での転送時暗号化

S3バケットポリシーによる強制 
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyHTTPRequests",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::example-bucket",
        "arn:aws:s3:::example-bucket/*"
      ],
      "Condition": {
        "Bool": {
          "aws:SecureTransport": "false"
        }
      }
    }
  ]
}

HTTPリクエストを拒否し、HTTPSアクセスのみを許可するバケットポリシーの例

  • aws:SecureTransport条件キーでHTTPSのみに制限
  • S3アクセスポイントも同様にHTTPSへ制限可能
  • CloudFrontと併用してオリジンアクセスアイデンティティを設定
  • S3アクセスログを有効化してセキュリティ監視を強化

API GatewayとALBの設定

API Gateway
  • カスタムドメイン名にACM証明書を設定
  • セキュリティポリシーはTLS 1.2以上のみを選択
  • リージョン型APIとエッジ最適化APIで証明書リージョンに注意
Application Load Balancer
  • HTTPをHTTPSにリダイレクトするリスナールールを作成
  • 最新のセキュリティポリシーを選択（ELBSecurityPolicy-TLS13-1-2-2021-06など）
  • ACMで管理された証明書をリスナーに関連付け
  • バックエンドEC2インスタンスとの通信もHTTPSに設定
HTTPリクエストをHTTPSへ
自動リダイレクトするだけで
ユーザー体験を損なわずに安全に！

VPN接続による保護

AWS Site-to-Site VPN
  • オンプレミスネットワークとAWS VPC間の暗号化された接続
  • IPsecプロトコルによる強力な暗号化
  • 2つの冗長トンネルによる高可用性
AWS Client VPN
  • リモートユーザーからVPCへの安全なアクセス
  • Active Directory、相互認証などの認証オプション
  • OpenVPNベースのTLS暗号化通信
VPNを使えばインターネット経由でも

安全な通信ができるよ！

実装手順

1. セキュリティグループで暗号化を適用

  • ALBやEC2インスタンスに対してHTTPS（ポート443）のみを許可
  • HTTP（ポート80）のインバウンドトラフィックをブロック
  • 最新の安全なプロトコルのみを許可するポリシーを設定

2. エッジサービスでの安全なプロトコル設定

  • CloudFrontでHTTPSを設定
  • セキュリティ要件に適したセキュリティポリシーを選択
  • HTTPからHTTPSへの自動リダイレクトを設定

3. VPN接続の設定

  • ポイントツーポイント接続にIPsec VPNを使用
  • ネットワーク間接続を保護し、データのプライバシーと整合性を確保
  • Site-to-Site VPNまたはClient VPNの適切な選択

4. ロードバランサーの安全なプロトコル設定

  • 最も堅牢な暗号スイートを提供するセキュリティポリシーを選択
  • Application Load BalancerにHTTPSリスナーを追加
  • ACMで管理された証明書を使用

5. その他のAWSサービスでの暗号化設定

  • Amazon RedshiftでSSL/TLS接続を必須に設定
  • S3バケットポリシーでHTTPSアクセスのみを許可
  • 各サービスの転送時暗号化機能を有効化
