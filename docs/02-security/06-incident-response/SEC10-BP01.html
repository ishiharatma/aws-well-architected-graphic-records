SEC10-BP01 重要な人員と外部リソースを特定する

AWS Well-Architected セキュリティの柱 - 準備

概要

  • 組織のインシデント対応体制を整備
  • 組織内外の担当者、リソース、法的義務を特定
  • 主要な担当者の連絡先情報と役割を明確化
  • 定期的な見直しと更新を実施
インシデント発生時に「誰に連絡すれば良いか」
迷わないように事前準備が大切！
緊急時の連絡網を整備しよう

期待される成果

  • 主要な担当者の最新リストを作成・維持
  • セキュリティイベント対応時の役割と責任を明確化
  • サードパーティのサービスプロバイダーやベンダーも含める
  • 適切な責任とアクセス権を持つ担当者による迅速な対応

外部パートナーの特定

  • ISV・パートナー・下請け業者の担当者
  • 適切なレベルのAWS Supportにサインアップ
  • 公的機関や規制当局への通知義務
  • すべての重要なソリューションプロバイダーとの取り決め
内部だけでなく外部の

一般的なアンチパターン

  • 主要担当者の最新情報をまとめたリストがない
  • 担当者や依存関係について全員が知っていると想定
  • インフラやアプリケーション設計のドキュメントがない
  • 新しい人員への適切なオンボーディングプロセスがない
  • 主要担当者が不在の場合のエスカレーションパスがない
「みんな知ってるはず」は危険！
明文化とプロセス整備で
確実な対応体制を構築しよう

実装のガイダンス

1. インシデント管理ソリューションの設定

  • セキュリティツール用アカウントにIncident Managerをデプロイ

2. 連絡先の定義

  • 連絡先ごとに最低2種類の連絡チャネル（SMS、電話、メールなど）

3. 対応計画の定義

  • インシデント発生時の対応要員を特定
  • 個人ではなく役割に合わせたエスカレーション計画を定義
  • 外部機関への情報提供担当者も含める
推奨AWSサービス
  • AWS Systems Manager Incident Manager
  • 主要連絡先の取得、対応プラン策定
  • オンコールスケジュールの自動化
  • エスカレーションプランの作成
