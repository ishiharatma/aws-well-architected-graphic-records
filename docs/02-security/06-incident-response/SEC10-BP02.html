  • 人事 (HR): 人的リソースの調整

  • 法務部: 法的コンプライアンス

RACIチャート例

アクティビティ インシデント管理者 技術チーム 経営陣 法務部

インシデント対応フロー

標準的なインシデント対応プロセス
1
検出 (Detection)
セキュリティイベントの発見・識別
2
分析 (Analysis)
インシデントの詳細調査・影響範囲の特定
3
封じ込め (Containment)
被害拡大の防止・一時的な対策
4
根絶 (Eradication)
脅威の完全除去・脆弱性の修正
5
復旧 (Recovery)
通常運用への復帰・監視強化
6
教訓 (Lessons Learned)
事後分析・プロセス改善
各段階で取るべき措置を
明確に定義することで
効率的な対応が可能に！

計画の構成要素

  • インシデント対応チームの概要: 目標と機能
  • 役割と責任: 利害関係者とその役割
  • コミュニケーションプラン: 連絡先と連絡方法
  • 通信手段のバックアップ: 帯域外通信の確保
  • 各段階と取るべき措置: 詳細な手順
  • 深刻度と優先順位: 分類とエスカレーション

AWS対応チームとサポート

AWS Support

AWS環境の計画、導入、最適化をサポート。サポートセンターが中心窓口。

AWS CIRT (カスタマーインシデント対応チーム)

24時間365日体制の専門チーム。アクティブなセキュリティイベント中にサポート。

AWS Shield

DDoS攻撃からウェブアプリケーションを保護するマネージド型防御サービス。

AWS Managed Services (AMS)

インフラ管理を継続提供。24時間365日でセキュリティ検出とアラート対応。

AWSの専門チームと連携して
より強固なインシデント対応を
実現できるよ！
