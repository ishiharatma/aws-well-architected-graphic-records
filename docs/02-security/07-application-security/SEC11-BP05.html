SEC11-BP05 パッケージと依存関係のサービスを一元化する

パッケージと依存関係の管理を一元化し、セキュリティリスクを軽減します。

公式ドキュメント

リスクレベル

高リスク

このベストプラクティスを活用しない場合、以下のリスクが発生する可能性があります：

  • 脆弱性のあるパッケージの使用
  • 悪意のあるコードの混入
  • 依存関係の可視性不足
  • パッケージ更新の管理困難
パッケージ管理の一元化は
セキュリティの基盤です！

期待される成果

承認されたパッケージのみが使用され、脆弱性が迅速に特定・修正される安全な開発環境を構築します。
