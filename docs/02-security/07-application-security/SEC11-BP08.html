SEC11-BP08 ワークロードチームにセキュリティ所有権を組み込むプログラムを構築する

開発チームがセキュリティに対する責任と所有権を持つ文化とプロセスを構築します。

公式ドキュメント

リスクレベル

中リスク

このベストプラクティスを活用しない場合、以下のリスクが発生する可能性があります：

  • セキュリティ責任の曖昧さ
  • セキュリティ意識の低下
  • セキュリティ問題の対応遅れ
  • 開発とセキュリティの分離

チーム全体の責任です！

開発チーム全体がセキュリティを自分事として捉え、プロアクティブにセキュリティ課題に取り組む組織文化を構築します。
